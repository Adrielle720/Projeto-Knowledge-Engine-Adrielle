Projeto: Knowledge Engine
Vamos construir um mecanismo de busca utilizando Lógica de Primeira Ordem. Esse sistema remete aos
primórdios da IA, que utilizava inferência lógica para responder perguntas em uma grande base de conhecimento.
O projeto consiste em montar uma base de conhecimento usando um dataset público e realizar perguntas.
O objetivo do projeto é trabalhar predicados e sentenças de forma prática, expandindo o ferramental para um
novo paradigma de programação.
Antes de explicar os requisitos, um pequeno tutorial de Prolog.

Novas Sentenças
O último exemplo pode ser reescrito através de duas sentenças:
c o n f r o n t o (TimeA , TimeB , Data , L o c a l ) :−
j o g o (TimeA , TimeB , _, _, Data , L o c a l ) ;
j o g o ( TimeB , TimeA , _, _, Data , L o c a l ) .
2
Onde: :− define uma sentença e ; é o operador ∨.
Obs: Existe o ¬, mas ele tem um significado um pouco diferente. True significa que o predicado foi realmente
afirmado e False significa a falha em provar o predicado com a base atual (isso não significa que o predicado é
falso)!

Em Prolog, podemos fazer sentenças que relacionam uma variável a um número:
empate (Time) :−
j o g o (Time, _, G, G, _, _) ;
j o g o (_, Time, G, G, _, _) .
pontos_empate (Time, 1 ) :−
empate (Time ) .
Toda variável passada como segundo argumento de pontos_time, receberá o valor 1.
Na sentença empate, ao usar a mesma variável G em dois argumentos, ele naturalmente iguala as duas, filtrando
apenas os predicados em que a quantidade de gols for igual.

O uso do predicado time é para não deixar a variável Time livre na função findall . O que faria com que os

O Projeto
Você deve procurar um dataset de interesse. Restrinja à apenas uma tabela-fonte do dataset original e selecione
entre 6 a 10 campos, mesclando variáveis qualitativas e quantitativas. Uma boa fonte de procura é o Kaggle.
Em seguida, deve fazer um script em Python para gerar a base de conhecimento e as perguntas no formato de
Prolog.
Após essa fase, você deve formular pelo menos 3 perguntas para o dataset. Essas perguntas precisam ser
respondidas com queries em Prolog.
Requisitos do Projeto:
• Individual (datasets não podem ser compartilhados - e nem usar o mesmo do tutorial).
• Data da Entrega: 3/Maio às 23h59 (esse prazo é improrrogável). Após o prazo, a nota de projeto será
desconsiderada.
• A entrega deve ser feita em um repositório GitHub. Você deve entregar o endereço do repositório no
Blackboard.
• Não serão aceitas implementações em outras linguagens além do ETL em Python e as queries em Prolog.
Rubrica:
• I: Não houve entrega, ou há apenas pedaços esparsos de código.
• D: A base de conhecimento foi montada mas não há 3 perguntas implementadas.
• C: Foram feitas 3 perguntas básicas, que poderiam ser respondidas com sentenças simples e diretas.
• B: Há pelo menos 1 pergunta mais sofisticada (envolvendo algum agrupamento ou técnica avançada).
• A: Todas as perguntas são sofisticadas.
4
pergunta básica = filtro direto; pergunta sofisticada = agregação, comparação, ordenação, composição de regras
ou uso de predicados auxiliares.
O Repositório precisa ter um README explicando todos os passos da construção do projeto e como rodá-lo.
A ausência do README implicará na perda de 1 conceito na nota.
Erros comuns em Prolog:
• Maiúscula vs Minúscula
• Esquecer do ponto final
• Fazer query em Program
• findall com variáveis livres
